How it works

Architecture

From an autonomous agent's decision to a cryptographically verifiable record — here is the full data flow.

🤖
Agent Action
e.g. PayBridge payment, policy decision, transfer.
→
🔐
SHA-256 Hash + Sign
Canonical record → deterministic digest.
content_hash = sha256(record)
→
Immutable storage + verification
🦭
Walrus Storage
Decentralized, replicated blob storage for the full log.
blob_id = PUT publisher/blob
⛓️
Sui Anchor
On-chain registry: content_hash → blob_id.
SuiRegistry { blob_id, hash }
→
✅
Verify
Anyone re-fetches the blob & compares the hash.
🔍
Verification: fetch(blob_id) → sha256(record) → compare to on-chain hash. If they match, the log is provably untampered — no trusted third party required.

Quick start

Store & verify in one command

# Python 3.11+ $ pip install httpx pydantic
$ export SUI_ADDRESS=0x6dca...your...address

# Store an audit log $ python -m walrus_audit_vault.cli store \
    --actor "0xpaybridge_agent" \
    --action "transfer_sui" \
    --decision "ALLOWED" \
    --amount 5000000000

✓ stored · blob_id = oXy...z9k · content_hash = 4f2c...e1a0

# Verify an audit log $ python -m walrus_audit_vault.cli verify --blob-id <blob_id>
✓ verified · on-chain hash matches
